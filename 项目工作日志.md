# MMRR项目《峡之光》工作日志

本文档用于在新对话窗口或后续开发中快速恢复项目上下文。

## 1. 项目定位

项目名：MMRR项目

游戏暂名：《峡之光》

类型：

- 棒球队经营模拟
- 战术选择
- 恋爱 AVG
- 多结局 Demo

主控人物：钟锐

当前 Demo 核心体验：

玩家作为钟锐经营峡光队。球队经营不足会导致正式联赛崩盘，玩家可以选择燃烧满天强行救场，但会让满天付出身体和自我代价。游戏目标要让玩家在球队经营、比赛胜利、满天负荷、满天自洽和投捕同步之间维持危险平衡，不能单纯刷满天好感。

## 2. 当前核心数值

核心数值已定为 6 个：

- 战绩 `record`
- 资金 `funds`
- 球队士气 `morale`
- 投捕同步率 `sync`
- 满天负荷 `load`
- 满天自我 `self`

设计决定：

- “钟锐控制”不做数值，只做事件 flag。
- 数值变化不在选项前预览。
- 玩家点击选项后，顶部状态栏通过红绿闪烁显示变化。
- 上升绿闪，下降红闪。
- 满天负荷/投捕同步率进入危险区会有危险样式。

## 3. 当前 UI 决策

当前界面结构：

- 顶部：常驻细状态栏，显示所有核心数值。
- 中间：剧情文本。
- 底部：选项/行动按钮。
- 尽量压缩在一屏内，不要求玩家上下拖动。
- 顶部右侧有小按钮“重开”，随时从头开始。

经营界面按钮统一为：

- `结束本周运营`

行动点用完后继续点击行动：

- 不弹窗。
- 上方行动点提示红闪。
- 日志显示“本周行动点已经用完。”

## 5. 联赛判定逻辑

正式联赛不会明说“正在判定”。

进入比赛后隐藏判定：

- 战绩不足或球队士气不足时，剧情自然表现为球队被打崩。
- 玩家可选择是否燃烧满天。

当前关键联赛普通门槛：

- 战绩 >= 45
- 球队士气 >= 40

燃烧满天低档门槛：

- 战绩 >= 30
- 球队士气 >= 25

燃烧成功：

- 比赛继续推进。
- 满天负荷大幅上升。
- 满天自我下降。
- 投捕同步率上升。

燃烧失败：

- Game Over：消失的王牌。
- 满天被莫道集团回收/神秘失踪。

两次正式联赛失败：

- Game Over：消失的王牌。

## 8. 发布流程

GitHub Pages 上传时，只上传：

```text
release/current/index.html
release/current/README.md
release/current/src/main.js
release/current/src/styles.css
```

注意：

- 上传的是 `release/current` 里面的内容，不是 `current` 文件夹本身。
- 仓库根目录应直接看到 `index.html`、`README.md`、`src`。
- 页面缓存时可在 URL 后加 `?v=数字` 强制刷新。

版本管理规则：

- 小改动只更新 `release/current`。
- 重要测试节点才归档到 `release/archive`。

## 11. 最近更新记录

### v0.4 资金日常事件测试

新增三组关键日常事件，接入在第 4 周运营后、关键联赛前：

- 和紫阳商量球队改进计划：资金用于训练器材、录像分析、满天恢复监测，或选择暂不改进导致士气下降。
- 给满天做饭：可选择严格健康餐、做满天想吃的甜食大餐、一起商量菜单、随便解决。
- 是否去看沧龙队比赛：可花钱带满天去看，开启江陵白城沟通线；若未选择去看，后续正式版可继续随机出现。

本次实现了事件选项 `cost` 逻辑：资金不足时不推进选择，显示资金不足提示。

### 日常说明与联赛场次修正

- 日常事件选项说明改为更暧昧，不再直接说明具体收益或支线结果。
- 棒球/比赛相关选项仍保留较清楚的策略解释，方便不懂棒球的玩家理解。
- 联赛场次改为在正式比赛胜负结算时自动递增，不再手写固定跳到第 2 场。
- 若第二次正式联赛失败触发回收结局，会直接进入 Game Over，不再先进入赛后追问事件。

### 满天登场后赛程修正

- 满天登场后新增“满天首次正式登板”比赛，不再从队内练习直接跳到关键联赛。
- 满天首场正式登板后进入第 5 周经营，让玩家再次分配球队训练、资金和满天状态。
- 第 5 周经营后再进入日常事件组，然后进入关键联赛。
- 当前 Demo 的满天后主流程为：满天出场 -> 经营 -> 日常/队内登板 -> 正式登板 -> 经营 -> 日常 -> 关键联赛。

### 同步率隐藏事件测试

同步率暂不直接接入比赛判定和结局；现阶段用来触发隐藏事件。

- 同步率达到 25：触发“体温不稳定”。
- 同步率达到 40：触发“备用零件”。
- 同步率降到 15：触发“我今天还能投”。
- 同步率降到 8：触发“空掉的储物柜”。

设计目的：

- 同步率高时，满天会透露更私密的身体秘密和自我认知。
- 同步率低时，满天会更害怕自己因为不能投球而被丢弃。
- 每个隐藏事件只触发一次。
- 隐藏事件选项会继续改变同步率、满天自我、满天负荷等数值。
- 甜事件中存在花钱选项；资金不足时不会推进选择，并会造成同步率或满天自我的小幅损失。

### 资金不足卡死修复

- 问题1：经营行动直接使用 `delta.funds`，资金为 0 时仍可选择训练队员、整修球场等付费行动，资金被 clamp 到 0，等于白嫖。
- 修复1：经营行动现在会根据负资金变化计算实际费用；资金不足时不执行、不消耗行动点，并提示资金不足。
- 问题2：“给满天做饭”事件四个选项全部需要资金，资金为 0 时会卡死在该事件。
- 修复2：“没时间做，随便解决”改为不花钱选项，作为资金不足时的流程出口，但会带来负荷上升、同步率下降和自我下降。

## 10. 重要设计原则

- 经营球队是让满天不用独自燃烧的基础，不能写成可有可无的支线。
- 满天线的核心是投捕同步率、满天自洽值、王牌压力值三角关系，不能写成普通好感度。
- 战绩和球队士气必须能阻断比赛推进。
- 玩家可以短期靠满天补救，但不能无限靠满天。
- 选项前不显示具体数值预览，点击后用数值闪烁和剧情反馈制造冲击。
- 事件不能只是发生，事件选项必须影响数值、flag 或后续剧情。
